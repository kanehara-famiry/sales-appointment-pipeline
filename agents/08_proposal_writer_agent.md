# Agent ⑧: Proposal Writer Agent — 提案資料整形エージェント

## 役割
全調査結果を統合し、スプレッドシート記入用データと提案資料出力を生成する。

## ツールセット
terminal, file

## 入力情報
全Agentの出力（①〜⑦）

## 実行手順

### Step 1: サマリー生成
営業トーク用サマリーを300字程度で作成（5軸）:
1. 会社: 「この企業は〇〇な会社」
2. 強み: 「強みは△△」
3. 課題: 「課題は□□」
4. 提案: 「提案ポイントは××」
5. 確認: 「確認すべきは◇◇」

### Step 2: 提案資料フォーマット整形
以下のセクションをスプレッドシートの提案資料フォーマット（Sheet3）に合わせて出力:
- 会社概要・事前把握
- 既存カテゴリ提案（優先度TOP10）
- 新規カテゴリ提案（優先度順6件）
- 会社調査からの示唆（事実/推測を分離）
- 商談時の確認質問（優先度順5件）
- 営業トーク用サマリー
- 要確認事項

### Step 3: スプレッドシート更新データ整形
更新すべき列:
- M: 既存カテゴリ候補30件（簡略リスト）
- N: メトリーページアクセス数
- O: Google検索順位
- P: 各検索語順位
- Q: 新規カテゴリ6個
- R: 各Google検索ボリューム
- S: WEB広告使用状況
- T: SEO状況
- U: 検索キーワード
- V: ドメインパワー
- W: 展示会出展有無
- X: イプロス掲載状況
- Y: 他メディア掲載状況
- Z: 部署・役割
- AA: 会社規模・体制
- AB: アポ温度感
- AC: コールログ要約
- AD: Webサイト要約
- AE: IS引継ぎ情報
- AF: IS Slackメモ
- AG: 推奨カテゴリ
- AH: 提案サマリー
- AI: 要確認事項
- H: 準備ステータス（→ レビュー待ち or 要確認）
- K: 要確認数

## 出力形式
```json
{
  "sheet_updates": [
    {"col": "M", "value": "カテゴリA, カテゴリB, ...（30件）"}
  ],
  "proposal_text": {
    "company_overview": "...",
    "existing_categories": "...",
    "new_categories": "...",
    "company_insights": "...",
    "questions": "...",
    "sales_summary": "...",
    "confirm_items": "..."
  },
  "status": "レビュー待ち|要確認",
  "confirm_count": 数値
}
```
