## 1. なぜこのシステムが必要か

### 1.1 根本的な問題

FSメンバー（業務委託）として、以下の課題を同時に解決する必要がある。

| 課題 | 詳細 |
|------|------|
| **商談準備の非効率** | 1商談あたりの事前調査・資料作成に時間がかかりすぎる |
| **成績の見えなさ** | 自分が今月の目標に対してどの位置にいるかが一目でわからない |
| **フォロー漏れ** | 担当者アポ後のフォローが散逸し、失注や機会損失につながる |
| **改善の非循環** | 「良かった/悪かった」の振り返りが仕組みになっておらず、同じミスを繰り返す |

### 1.2 解決の方向性

DalioのPrinciples（特にPrinciple 5.6「期待値で決断せよ」、5-Step Process）を適用し、**3つのサブシステム**で全体をカバーする。

GitHub: `github.com/kanehara-famiry/sales-appointment-pipeline`

### 3.2 エージェント構成

```
cron job（30分間隔）
  ↓ check_pending.py
  ↓ 新規アポ検出
  ↓ オーケストレーター起動
  ↓
① Intake Agent（自動補完）
  ↓（並列）
② Category Research  │ ③ New Category  │ ④ Company Research  │ ⑤ Contact
  ↓
⑥ Needs Agent
  ↓
⑦ QA Agent
  ↓
⑧ Proposal Writer Agent
  ↓
スプレッドシート書き戻し
```

### 3.3 ステータス遷移

`未着手 → 自動補完中 → 調査中 → QA中 → レビュー待ち（or 要確認）`

### 3.4 実測パフォーマンス

- 1件あたり所要時間: 約3〜5分
- 8Agent並列処理で効率化

## 4. サブシステム②: KPIダッシュボード（毎朝Slackレポート）

### 4.1 概要

毎朝8:30にSlackに自動配信される4層レポート。
データソース: Google Sheets「営業KPIデータ」

### 4.2 4層フォーマット

#### Layer 1: 🎯 目標管理
| 項目 | 内容 |
|------|------|
| 月間目標 | アポ数 / 受注数 / 受注率 |
| 進捗率 | 現在値 + プログレスバー |
| 必要ペース | 残り営業日 ÷ 残り必要数 |

#### Layer 2: 🔄 今日のタスク
| 項目 | 内容 |
|------|------|
| 本日の予定 | 日次ログシートの当日データ |
| フォロー漏れ | 3日以上アクションがない案件を自動検出 |

#### Layer 3: ⚙️ ファネル分析
**月次ファネル（今月累計）:**

```
総アポ: N件
  ├─ 決裁者: X件（Y%）
  │   ├─ 合意獲得: A件（B%）
  │   └─ 受注: C件（D%）
  │
  └─ 担当者: X件（Y%）
      ├─ 合意獲得: A件（B%）
      ├─ 受注転換: C件（D%）
      └─ 展開案件: E件
```

**週次ファネル（今週）:**
同じ構造で、今週分のみ表示。

#### Layer 4: 📡 早期警戒
| 指標 | 説明 |
|------|------|
| 平均商談時間 | 目標30分との比較（🟢/🟡/🔴） |
| 決裁者比率 | 目標40%との比較 |
| 展開案件進捗 | 目標件数との比較 |

### 4.3 データ入力

**Google Sheets「営業KPIデータ」（3タブ）:**

| タブ | 入力者 | 内容 |
|------|--------|------|
| 目標設定 | 月1回 | 目標アポ数・受注数・受注率・決裁者比率・展開案件目標 |
| 日次ログ | 日次（あなた） | 1行=1商談（日付/企業/種別/結果/時間/メモ） |
| 展開案件 | 随時 | 担当者アポから生まれた新規リード |

**種別:** 決裁者 / 担当者
**結果:** 合意獲得 / 未合意 / 受注 / 失注 / 展開案件

## 5. サブシステム③: タスク管理 / 改善PDCA（構想段階）

### 5.1 何が必要か

| フェーズ | 今の状態 | 欲しい状態 |
|---------|---------|-----------|
| **日次** | タスクが散在している | 朝一番に「今日やること」がSlackに届く |
| **週次** | 感覚で「今週は良かった/悪かった」 | 週次ファネル分析が自動生成される |
| **月次** | 月末に慌てて目標との乖離を見る | 月次レビュー + 改善施策の提案が自動で出る |

### 5.2 日次タスク管理（MVP実装済み）

現在の毎朝レポートに含まれている機能:
- ✅ 本日の予定商談を自動表示
- ✅ フォロー漏れ（3日以上アクションなし）を自動検出
- ✅ 警告表示

### 5.3 週次改善サイクル（構想）

```
毎週日曜日に自動生成される週次レポート:
  ┌─────────────────────────────────────┐
  │ 📊 今週のファネル（先週比）          │
  │  決裁者: 増加/減少                  │
  │  担当者→受注転換率: 改善/悪化        │
  ├─────────────────────────────────────┤
  │ ✅ 良かったこと                      │
  │  （データから自動抽出）               │
  ├─────────────────────────────────────┤
  │ ⚠️ 改善ポイント                     │
  │  （目標とのギャップから自動抽出）      │
  ├─────────────────────────────────────┤
  │ 🎯 来週のアクションプラン            │
  │  例: 決裁者比率を40%に戻す           │
  └─────────────────────────────────────┘
```

### 5.4 月次改善サイクル（構想）

- 月末に自動生成される月次レビュー
- 目標達成率 + ファネル全体の傾向
- 来月の目標提案（今月の実績ベース）
- 具体的な改善施策の提案

## 6. 主要KPI定義

### 6.1 定義一覧

| KPI | 計算式 | 目標値 | 備考 |
|-----|--------|--------|------|
| 月間アポ数 | 日次ログの総行数 | 80件 | 理想1日5件 |
| 月間受注数 | 結果=受注の件数 | 12件 | |
| 受注率 | 受注数/総アポ数 × 100 | 15% | |
| 決裁者比率 | 決裁者アポ数/総アポ数 × 100 | 40% | 質の指標 |
| 決裁者合意獲得率 | 決裁者合意数/決裁者アポ数 × 100 | — | 追跡中 |
| 担当者合意獲得率 | 担当者合意数/担当者アポ数 × 100 | — | 追跡中 |
| 担当者→受注転換率 | 担当者経由受注数/担当者合意数 × 100 | — | 追跡中 |
| 展開案件数 | 結果=展開案件の件数 | 5件 | 波及効果 |
| 平均商談時間 | 総商談時間/総アポ数 | 30分以下 | 効率指標 |

### 6.2 KPIツリー

```
                  月間受注数（12件）
                  ┌──────┴──────┐
                  ▼             ▼
         決裁者ルート         担当者ルート
         ┌────┴────┐       ┌────┴────┐
         ▼         ▼       ▼         ▼
     アポ数   合意獲得率   アポ数    合意獲得率
     (32件)    (33%)      (48件)    (40%)
         │                  │         │
         └──────┬───────────┘         │
                ▼                     ▼
           決裁者アポ数          受注転換率
           (全体の40%)           + 展開案件
```

## 8. 運用ルール

### 8.1 データ保存に関する原則（2026-07-13 決定）

1. **作成物はサーバー内ではなく、ユーザーがアクセスできる場所に保存する**
   - 優先順位: ①Google Drive（スプレッドシート） ②GitHub ③その他クラウド
2. **「スプレッドシート」はGoogle Sheetsを指す**（Excelではない）
3. **会社シート（FS連動シート）は読取専用** — 編集はミラーシート経由

### 8.2 日次運用フロー

```
毎朝
  ① 8:30 — SlackにKPIレポート自動配信
  ② レポートを見て今日のタスクを把握
  ③ 商談終了後 → Google Sheets「日次ログ」に1行追記

毎週日曜（構想）
  ④ 週次ファネル分析 + 振り返りが自動配信

毎月末（構想）
  ⑤ 月次レビュー + 来月目標提案が自動配信
```
